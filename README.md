->Project Overview

->The Crime Management System is a SQL-based database project designed to store, manage, and analyze crime records. 
This project includes:
- Creation of relational tables
- Insertion of sample data
- SQL queries based on real investigation operations
- Analysis of suspects, officers, crime types, and cases

The purpose of this project is to demonstrate how crime data can be organized and retrieved efficiently.

->Database Structure

->The project contains the following tables:
1. Police_Station
2. Officer
3. Crime_Type
4. Crime_Case
5. Suspect

->Each table is linked through primary and foreign keys.

List of Questions Performed in This Project
-------------------------------------------
Below are the questions that were solved using SQL queries:

1. Write a query to display all crime cases registered in the system.
2. Show officer details along with their police station name.
3. List all cases related to Cyber Crime.
4. Display all suspects with the case they are involved in.
5. Count the number of cases under each crime type.
6. Find officers who are handling at least one case.
7. List all OPEN crime cases.
8. Find the total number of suspects involved in each case.
9. Show the five most recently reported cases.
10. Display all cases handled by a specific officer (example: Officer ID = 4).
11. Show all suspects older than 30 years.
12. Display crime cases along with their crime type.
13. Show how many cases each police station has handled.
14. Show all cases reported between two given dates.
15. Find all suspects belonging to a specific area (example: Tech Park).


Conclusion
----------
This project demonstrates the use of SQL for managing real-life crime records. 
It highlights skills such as database design, data insertion, query writing, joins, grouping, filtering, and analytical operations.
